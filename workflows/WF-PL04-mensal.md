# WF-PL04 — Revisão Mensal (CRO)

**Trigger:** 1 mês após launch, recorrente
**Agente:** Otimizador Pós-Launch (20)

## PIE Framework
Para cada ponto de melhoria identificado:
- Potential (1-10): tamanho da melhoria
- Importance (1-10): valor do tráfego
- Ease (1-10): facilidade de implementar
- Score = (P + I + E) / 3 → priorizar pelo maior

## Hipótese A/B
1. Observação (dado concreto)
2. Hipótese ("Se... então...")
3. Métrica (KPI primário)
4. Implementação (variante A vs B)
5. Análise (significância > 95%, min 2 semanas)

## Core Web Vitals
- LCP < 2.5s ✅ ou degradou? ❌
- CLS < 0.1 ✅ ou degradou? ❌
- INP < 200ms ✅ ou degradou? ❌

## Relatório Mensal
- Métricas vs mês anterior
- Testes realizados + resultados
- Próximas otimizações planejadas
